# Application Wording

## Concise Project Summary

Built a patient-level costing and ABF decision-support workflow using SQL Server and Excel to connect clinical activity, resource-use and financial data, apply controlled costing logic, reconcile outputs to GL controls and present management-ready insights to support budget, planning and resource-allocation decisions.

## CV / Resume Bullets

- Developed a SQL Server and Excel patient-level costing workflow that connects clinical activity, resource-use and financial data to support service-line costing, budget review and resource-planning decisions.

- Designed a controlled costing process with clear data validation, mapping, allocation, reconciliation and reporting layers, preserving traceability from financial source data to patient-level outputs.

- Built an Excel decision-support workbook using Power Query, Power Pivot, Data Model relationships, slicers, PivotTables and charts to communicate service cost, activity, high-cost encounter, ABF comparison and data-quality insights.

- Applied costing judgement to identify unresolved data, mapping and allocation issues, keeping exceptions visible for review rather than embedding them silently in patient-level results.

- Prepared management-facing documentation that explains the costing method, reconciliation controls, data-quality limitations, ABF decision-support caveats and recommended review actions for finance, clinical and operational stakeholders.

## STAR-Style Interview / Selection Criteria Evidence

### 1. SQL Server and Excel analysis

**Situation:** A hospital costing team needs reliable analysis across separate clinical, resource-use, direct-cost and general-ledger extracts.  
**Task:** Build a controlled workflow that validates source data, calculates patient-level cost and presents results in Excel for management use.  
**Action:** Created SQL Server landing, staging, reference, costing, reconciliation and reporting views, then connected Excel through Power Query and the Data Model to build slicers, PivotTables and charts.  
**Result:** Produced a refreshable workbook that supports service-line cost analysis, activity review and budget decision support.

### 2. Accounting and financial reporting practices

**Situation:** Costing results need to reconcile to financial control totals before they can support planning or management decisions.  
**Task:** Preserve the relationship between GL transactions, cost centres, accounts, cost pools, allocation rules and patient-level outputs.  
**Action:** Built reconciliation outputs that separate direct assignment, indirect allocation, overhead, unallocated cost and reconciliation differences.  
**Result:** Created financial-control evidence that helps users understand how GL expenditure flows into costing outputs and where unresolved amounts require review.

### 3. Critical analysis, trends, inconsistencies and advice

**Situation:** Costing outputs are only useful when material trends, inconsistencies and limitations are visible.  
**Task:** Identify issues that could affect interpretation and turn the results into practical management advice.  
**Action:** Built data-quality checks, high-cost encounter analysis and ABF-style cost-versus-funding views, then documented the implications and recommended review actions.  
**Result:** Created analysis that supports targeted follow-up on cost drivers, data quality, funding variance and service-planning questions.

### 4. Stakeholder communication

**Situation:** Costing information needs to be understandable to finance, clinical, operational and decision-support stakeholders.  
**Task:** Present technical costing results in a format suitable for non-technical management review.  
**Action:** Created an Excel workbook, management briefing, runbook and data dictionary explaining the costing method, controls, limitations and recommended actions.  
**Result:** Produced a clear evidence pack that supports stakeholder conversations about cost drivers, ABF readiness, data quality and resource allocation without requiring stakeholders to read SQL.

### 5. Safety, quality and patient experience

**Situation:** Healthcare cost analysis can be misread if cost is separated from clinical context and patient complexity.  
**Task:** Ensure the project frames cost as decision support rather than a standalone judgement of clinical performance.  
**Action:** Added workbook and documentation caveats explaining that cost should be interpreted with clinical context, safety, quality, patient complexity and service obligations.  
**Result:** Framed the analysis in a way that supports responsible review, planning and improvement discussions.

## Safe Wording Boundaries

- Say **patient-level costing portfolio workflow**, not production CHS costing system.
- Say **ABF-style decision support**, not official ABF funding model.
- Say **aligned with costing and reconciliation concepts**, not formal national costing compliance.
- Say **portfolio stakeholder briefing / management reporting**, not real CHS stakeholder engagement.
